#### Что было сделано

- Вчера весь день проверял PR коллег, искал best practices, помогал им писать код и мыслить в рамках React.
- Настроил защиту ветки `develop` и пересмотрел защиту для `main` (теперь нужно 2 approvement вместо одного).
- Подключил Material UI.
- Провели третье собрание для решения возникшей проблемы.
- В ходе работы выбор Workflow остался за Git Flow, его придерживаться и будем (у нас нет возможности заранее писать
  тесты, чтобы не сломать main, поэтому работать через связку develop -> release -> main намного безопаснее).

#### Проблемы

Не все в нашей команде имеют опыт работы с React, но очень хотели активно начать писать код, что привело к ситуации,
в которой пытались написать множество компонентов UI-обёрток, заключающих в себе только один HTML-тег и стили.
Столкнулись с проблемой, в которой хотели написать с нуля свою UI-библиотеку, не имея опыта.

#### Результаты собрания

- Было решено использовать Material UI вместо написания своих микро-компонентов с нуля. Теперь каждый будет работать над
  своими features, не опираясь на shared-компоненты (так как у всех будут одинаковые готовые базовые строительные блоки).
  Мне предстоит пройти собеседование, поэтому я не могу в данный момент активно работать над проектом с командой, хотя и
  стараюсь делать все, что в моих силах. Надеюсь, что с Material UI отпадут конфликты в борьбе за shared-компоненты
  и все смогут комфортно работать над своей часть проекта.
- Структуру проекта в плане папок в целом утвердили.
- Предстоит настроить Deployment-pipeline через GitHub Actions.
