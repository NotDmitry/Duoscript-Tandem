#### Что было сделано

- Наконец-то настроил AppRouter, добавил к нему шаблоны основных страниц, теперь команде будет с чем работать.
- Настроил CI/CD pipeline: CI через GitHub Actions запускает ESLint при PR и Push в `main` и `develop`. CD пока через

#### Проблемы

- Основная проблема связана с deploy на GitHub Pages. Для него нужно настраивать `baseUrl` в Vite, а это ломает навигацию.
  Обойти это ограничение можно, если заняться дополнительной настройкой и подстановкой `baseUrl` в разных окружениях
  (prod/deploy), но более простым решением оказалось использовать Netlify. За считанные минуты создался автоматический
  deploy, подтянулись скрипты для Vite, а также при создании PR создаётся Preview-версия, так что работу можно
  проверить ещё до одобрения PR на слияние с `main`. Рекомендую.
- Разработчики react-router выдумали целых 3 способа использования библиотеки (которая теперь и библиотека,
  и фреймворк). Будем придерживаться классического варианта (declarative mode), который соответствует классическому
  react-router-dom v6 (до v6.4).
